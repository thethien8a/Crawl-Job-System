Yêu cầu chức năng JobSearchWeb

Ứng dụng tìm kiếm việc làm ngành dữ liệu - The Data Index

1. Tìm kiếm việc làm

Tìm kiếm thông minh kết hợp từ khóa song ngữ, fuzzy matching và full-text search trên dữ liệu từ Supabase.

  • FR-SEARCH Nhập từ khóa
    • Tìm trong tiêu đề, công ty, địa điểm, từ khóa chuyên môn
    • Giới hạn độ dài tối đa 100 ký tự
  • FR-SEARCH Mở rộng từ khóa song ngữ
    • intern - thực tập - trainee
    • fresher - junior - mới tốt nghiệp
    • senior - lead - trưởng nhóm
    • manager - head - trưởng phòng
    • remote - từ xa - online
  • FR-SEARCH Hỗ trợ sai lỗi chính tả
    • Fuzzy matching dùng pg_trgm (ví dụ: Data Analysts ra Data Analyst)
  • FR-SEARCH Full-text search
    • Tối ưu truy vấn văn bản sâu trên tiêu đề công việc
  • FR-SEARCH Sắp xếp theo độ liên quan
    • Ưu tiên kết quả full-text match
    • Bổ sung similarity và thứ tự hạn nộp / tiêu đề

2. Lọc việc làm

Lọc kết quả tìm kiếm theo địa điểm, nguồn tuyển dụng và kết hợp với từ khóa.

  • FR-FILTER Lọc theo địa điểm
    • Dropdown danh sách địa điểm lấy từ database
    • Tìm kiếm không dấu / chữ hoa thường
  • FR-FILTER Lọc theo nguồn tuyển dụng
    • Dropdown TopCV / ITviec / VietnamWorks
  • FR-FILTER Kết hợp bộ lọc
    • Cho phép tìm kiếm đồng thời keyword + location + source_site
  • FR-FILTER Xóa bộ lọc
    • Nút reset toàn bộ điều kiện tìm kiếm

3. Hiển thị kết quả

Trình bày danh sách việc làm dạng thẻ, hỗ trợ phân trang và hiển thị chỉ số tổng quan.

  • FR-RESULT Hiển thị thẻ việc làm
    • Tiêu đề công việc (link nguồn)
    • Tên công ty
    • Địa điểm làm việc
    • Nguồn đăng tuyển
    • Từ khóa chuyên môn từ tiêu đề
    • Hạn nộp hồ sơ
  • FR-RESULT Phân trang
    • Mặc định 12 việc / trang
    • Tối đa 100 việc / trang theo API
  • FR-RESULT Chỉ số tổng quan
    • Tổng số vị trí tìm được
    • Tổng số công ty
    • Số địa điểm có việc làm
  • FR-RESULT Trạng thái không có kết quả
    • Thông báo gợi ý thay đổi bộ lọc

4. Chi tiết việc làm

Hiển thị thông tin chi tiết của một tin tuyển dụng khi người dùng chọn xem.

  • FR-DETAIL Xem chi tiết theo URL
    • Decode URL từ route parameter
    • Trả 404 nếu không tìm thấy việc làm
  • FR-DETAIL Hiển thị thông tin
    • Tiêu đề công việc
    • Tên công ty
    • Địa điểm
    • Mức lương (nếu có)
    • Yêu cầu kinh nghiệm
    • Hạn nộp hồ sơ
  • FR-DETAIL Chuyển hướng tin gốc
    • Nút "Xem tin gốc" mở tab mới an toàn
  • FR-DETAIL Quay lại trang chủ
    • Link back giữ nguyên trạng thái tìm kiếm

5. Gợi ý việc làm theo CV

Người dùng upload CV (PDF/DOCX) để nhận Top 10 việc làm phù hợp nhất dựa trên vector embedding và kỹ năng trùng khớp.

  • FR-REC Upload CV
    • Chấp nhận định dạng PDF và DOCX
    • Giới hạn kích thước tối đa 5MB
    • Giới hạn tối đa 5 trang
    • Hỗ trợ kéo-thả (drag-drop)
  • FR-REC Chọn bộ lọc nghiệp vụ
    • Chọn địa điểm làm việc mong muốn
    • Nhập số năm kinh nghiệm
  • FR-REC Xử lý CV phía backend
    • Trích xuất text thô từ PDF (pdfplumber) / DOCX (python-docx)
    • Làm sạch text
    • Trích xuất kỹ năng theo taxonomy (8 nhóm require_*)
    • Embed CV bằng Gemini embedding (task_type=RETRIEVAL_QUERY, 768 chiều, L2-normalize)
  • FR-REC Truy vấn Qdrant
    • Query vector embedding CV
    • Filter cứng: job chưa hết hạn, địa điểm, số năm kinh nghiệm phù hợp
    • Lấy top N (ví dụ 50) kết quả đầu tiên
  • FR-REC Re-rank hybrid
    • final = 0.6 * cosine + 0.4 * skill_overlap
    • Trọng số nhóm skill: programming_languages, frameworks, tools, cloud_skills, knowledge, domain_knowledge
  • FR-REC Hiển thị kết quả Top 10
    • Danh sách 10 việc làm phù hợp nhất
    • Badge % phù hợp (score)
    • Hiển thị các kỹ năng trùng khớp (matched_skills)
  • FR-REC API gợi ý
    • POST /api/v1/recommend nhận multipart/form-data (file, locations, years_experience)
  • FR-REC Bảo mật & vận hành
    • Rate-limit riêng chặt hơn endpoint /jobs
    • Lưu CV gốc lên S3 bucket riêng, lifecycle tự xóa sau 30 ngày
    • Không log nội dung CV
    • Không cần đăng nhập, mỗi lần upload là phiên độc lập

6. Giao diện người dùng

Thiết kế giao diện mang phong cách academic/editorial, responsive và tối ưu trải nghiệm.

  • FR-UI Giao diện tổng thể
    • Bố cục hero search + kết quả bên dưới
    • Màu sắc Navy Academic / Paper Ivory / Brass Gold
    • Font Playfair Display + Plus Jakarta Sans
  • FR-UI Responsive
    • Hoạt động tốt trên desktop, tablet, mobile
  • FR-UI Trạng thái tải dữ liệu
    • Loading spinner khi chờ API
    • Thông báo lỗi khi API thất bại

7. Hệ thống & API

Backend FastAPI cung cấp API đọc dữ liệu từ Supabase, hỗ trợ caching, rate-limiting và bảo mật cơ bản.

  • FR-SYS API tìm kiếm
    • GET /api/v1/jobs hỗ trợ keyword, location, source_site, page, page_size
  • FR-SYS API dropdown dữ liệu
    • GET /api/v1/jobs/locations
    • GET /api/v1/jobs/source-sites
  • FR-SYS API thống kê
    • GET /api/v1/jobs/stats trả total_jobs, total_companies, jobs_by_location
  • FR-SYS API chi tiết
    • GET /api/v1/jobs/{job_url}
  • FR-SYS Caching
    • Dropdown cache 1 giờ
    • Search/stats cache 1 phút
  • FR-SYS Giới hạn request
    • Rate limit theo IP (slowapi)
  • FR-SYS Bảo mật cơ bản
    • CORS chỉ cho phép origin được cấu hình
    • API read-only (chỉ cho phép GET)
    • Ẩn chi tiết lỗi nội bộ ở production
  • FR-SYS Health check
    • GET /health kiểm tra kết nối database
Ghi chú: Các yêu cầu trên được trích xuất từ mã nguồn JobSearchWeb (backend FastAPI, frontend React/Vite, cơ chế tìm kiếm hybrid, bảo mật cơ bản) và tài liệu thiết kế gợi ý việc làm theo CV (Gemini embedding + Qdrant + hybrid scoring). Có thể đưa trực tiếp vào phần yêu cầu chức năng của báo cáo đồ án tốt nghiệp.
