CV Recommendation Flow · MVP-first

User submit CV PDF thì backend làm gì?

File này mô tả luồng request thật phía sau nút submit: browser gửi CV, backend parse toàn bộ nội dung CV, extract skill kể cả trong project cá nhân, gọi OpenRouter embedding, search Qdrant, fallback nếu filter quá hẹp, rồi re-rank bằng semantic + skill + experience + location.

Không lưu CV gốc trong MVP Location/exp do user nhập Skill extract trên toàn bộ CV Qdrant chỉ là serving index
Công thức rank cuối final = 0.50 * cosine + 0.30 * skill_overlap + 0.12 * experience_match + 0.08 * location_match
Sơ đồ tổng quan dễ nhìn Submit CV → Top 10 jobs
Browser / React UI Client
1

User chọn CV PDF/DOCX

Frontend kiểm tra nhanh đuôi file, dung lượng <= 5MB và hiển thị form chọn location + số năm kinh nghiệm.

2

Submit form

React gửi multipart/form-data tới POST /api/v1/recommend: file CV, locations, years_experience.

12

Nhận Top 10

UI render badge % phù hợp, matched skills, location, salary, company và link job_url để user mở JD.

FastAPI Backend Serving
3

Validate request

Backend kiểm tra định dạng PDF/DOCX, size, số trang. Nếu CV scan/ảnh không extract được text thì trả lỗi rõ ràng.

4

Parse CV thành text

Dùng pdfplumber/python-docx để lấy text thô. MVP xử lý trong memory, không upload CV gốc lên S3.

5

Clean text

Chuẩn hóa whitespace/ký tự lỗi để text ổn định hơn trước khi extract skill và embed.

6

Extract skills toàn CV

Chạy taxonomy trên toàn bộ CV text, không chỉ mục Skills. Vì vậy skill trong project/work experience vẫn được bắt.

7

Embed CV query

Gọi OpenRouter openai/text-embedding-3-small với input type search_query, output 768 chiều và L2-normalize.

Quyết định quan trọng

Location/experience không parse từ CV. User tự chọn trên UI vì nơi muốn làm và số năm ứng tuyển có thể khác thông tin trong CV.

10

Re-rank hybrid

Backend tính lại score từ Qdrant cosine, skill_overlap, experience_match và location_match. Sau đó sort giảm dần.

11

Dựng explanation

Trả matched_skills theo từng nhóm. Nếu matched skills rỗng, dùng explanation fallback: phù hợp theo nội dung CV/JD.

Qdrant + Index có sẵn Search
0

Job đã được index trước

Lakehouse-Lite đọc Silver parquet, build job text, embed bằng search_document, upsert vào collection jobs_v2.

8

Strict query

Search vector với filter: job chưa hết hạn, location user chọn, min_exp_level <= years_experience hoặc null.

?

Nếu strict quá ít kết quả

Backend query fallback rộng hơn: giữ deadline filter nhưng bỏ/nới location-exp. Sau đó re-rank để job phù hợp vẫn lên đầu.

9

Trả candidate jobs

Qdrant trả top 50-100 job gồm vector score và payload: job_url, title, company, location, exp, salary, require_* skills.

Luồng request theo thời gian

T-0

Trước khi user submit, Qdrant đã có job vectors từ Silver parquet. Đây là lý do request realtime không cần đọc S3/Supabase.

0-2s

Validate file, parse PDF/DOCX, clean text. Nếu không có text vì CV scan/ảnh, trả lỗi sớm.

2-5s

Extract skills toàn CV và gọi Gemini để tạo embedding query.

5-8s

Search Qdrant strict. Nếu ít kết quả thì fallback query rộng hơn.

8-10s

Re-rank, dựng explanation, trả Top 10 cho frontend.

Input/output của từng lớp

  • Frontend gửi: CV file, locations optional, years_experience optional.
  • Parser tạo: full_cv_text đã clean.
  • Taxonomy tạo: CV skills theo nhóm require_programming_languages, frameworks, tools, cloud, knowledge...
  • Gemini tạo: cv_vector 768 chiều dùng để query.
  • Qdrant trả: candidate jobs + cosine score + payload.
  • Re-ranker trả: Top 10 đã sort + score cuối + matched_skills để giải thích.
