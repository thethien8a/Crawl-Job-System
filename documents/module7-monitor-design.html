Hạ tầng Monitor dùng chung

Caddyfile

Cổng basic_auth duy nhất; reverse_proxy /business/* → dashboards và /grafana/* → grafana; HTTPS tự động.

prometheus.yml

entrypoint.sh render ${VAR} env (AIRFLOW_STATSD_EXPORTER_TARGET, SUPABASE_PROJECT_REF, SUPABASE_METRICS_SECRET_KEY) trước khi khởi động.

Grafana provisioning

datasources.yml (Prometheus uid) + dashboards.yml (folder lakehouse-lite, file provider, reload 30s).

EFS chia sẻ

business/reports/ mount EFS tới container dashboards nginx; volume dữ liệu Prometheus & Grafana độc lập.

Hai luồng giám sát

Vận hành (time-series qua Prometheus, cảnh báo theo thời gian thực) tách biệt nghiệp vụ (snapshot HTML Altair, chụp chất lượng dữ liệu Bronze/Silver mỗi lần DAG chạy).

Pull model

Prometheus chủ động scrape 3 target (self, Airflow qua statsd-exporter, Supabase Metrics API) theo lịch 15s/60s — không push, dễ debug target down.

Dashboard as code

Datasource và dashboard Grafana được provisioning bằng file YAML + JSON trong repo, reload 30s, không cần config tay trên UI.

Single-login edge

Caddy bao basic_auth + HTTPS ở biên; Grafana đặt anonymous Viewer, nginx không auth — service nội bộ ẩn sau mạng Docker.

Business dashboards

Altair chart sinh bởi Airflow DAG (generate_bronze/silver_dashboard), ghi HTML vào EFS, nginx serve tĩnh — không cần server Python.

Prometheus TSDB

Lưu chuỗi thời gian 15s (Supabase 60s) trên ổ cứng EC2; truy vấn PromQL trong Grafana, có thể bật alert rule khi DAG fail hoặc DB tăng trưởng bất thường.
