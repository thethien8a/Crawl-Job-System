Hạ tầng Supabase dùng chung

Kết nối

psycopg2.connect qua get_connection() — host/port/user/password từ biến môi trường.

Schema do JobData định nghĩa

Dataclass JobData → kiểu PG tự sinh (list[str] → TEXT[]), CREATE TABLE IF NOT EXISTS + migration.

Conflict key

unique_url làm khóa UPSERT — chạy lại không trùng lặp.

Commit từng site

Mỗi site nạp & commit riêng: site hỏng rollback lô của nó, không chặn các site còn lại.

OLTP hàng

PostgreSQL lưu theo hàng, tối ưu đọc/ghi theo dòng và phục vụ API web — đối lập với Gold columnar dành cho phân tích tổng hợp.

UPSERT trên unique_url

Khóa xung đột là unique_url (chuẩn hóa URL: cắt query/fragment, ITviec bỏ -digits); UPSERT nên chạy lại an toàn, không sinh trùng.

Ánh xạ clean → tên gốc

clean_job_title → job_title, clean_location → location, clean_company_name → company_name: frontend nhận giá trị đã làm sạch mà schema DB vẫn giữ tên cột gốc.

Schema do JobData định nghĩa

Target schema do dataclass JobData quy định; kiểu PG tự sinh (list[str] → TEXT[]), kèm migration thêm cột unique_url và xoá dòng stale.

Nạp theo lô + commit từng site

Dùng execute_values với BATCH_SIZE = 100 để giảm round-trip; commit theo site cho tiến độ từng phần khi một site lỗi.

Reader dùng chung

Dùng lại get_jobs_silver_by_site (Polars, lấy file mới nhất theo LastModified) — cùng reader với tầng Silver và Gold.
