\section{Module 5: Giám sát dữ liệu (Monitoring Layer)}
\label{sec:monitoring}

Module giám sát chia làm hai nhóm: \textbf{business dashboard} (Python sinh HTML
tĩnh từ S3) và \textbf{infrastructure monitoring} (Prometheus + Grafana).
Cả hai được phục vụ qua Caddy reverse proxy với single-login.

\begin{longtable}{p{3.5cm} p{3cm} p{7.5cm}}
\toprule
\textbf{Nhóm} & \textbf{Công nghệ} & \textbf{Mục đích} \\
\midrule
\endhead
Business dashboard & Python + Altair + nginx & HTML tĩnh từ S3 Bronze/Silver, serve qua nginx \\
Infrastructure & Prometheus + Grafana & Metrics Airflow (StatsD) + Supabase API \\
Reverse proxy & Caddy & Single-login, TLS, route /business/ + /grafana/ \\
\bottomrule
\end{longtable}

\subsubsection{Bronze dashboard}

File \texttt{bronze\_dashboard.py}
đọc danh sách S3 objects trong Bronze bucket, thống kê dung lượng và số bản
ghi, rồi sinh 3 chart Altair (Hình~\ref{fig:bronze-dashboard}):

\begin{lstlisting}[style=pystyle, caption={Bronze dashboard — 3 chart chính}]
def build_chart_blocks(frame: pl.DataFrame) -> list[tuple[str, str, str]]:
    if frame.is_empty():
        return []
    return [
        ("bronze-storage-by-site", "Total Bronze storage by site",
         chart_to_json(build_storage_by_site_chart(frame))),
        ("bronze-daily-storage", "Bronze storage by date and source site",
         chart_to_json(build_daily_storage_chart(frame))),
        ("bronze-daily-records", "Record count by date and source site",
         chart_to_json(build_daily_record_count_chart(frame))),
    ]
\end{lstlisting}

Metric cards hiển thị \texttt{BronzeSummary}: tổng dung lượng (MB), số source
site, số object. Chart dùng Altair \texttt{mark\_bar} (bar chart theo site) và
\texttt{mark\_line} (line chart theo ngày), serialize thành JSON nhúng vào HTML.

\subsubsection{Silver dashboard}

File \texttt{silver\_dashboard.py}
đọc Silver parquet từ S3, sinh 4 bảng review chất lượng dữ liệu
(Hình~\ref{fig:silver-dashboard}):

\begin{lstlisting}[style=pystyle, caption={Silver dashboard — 4 bảng review}]
def build_table_sections(frame: pl.DataFrame, max_table_rows: int) -> list[str]:
    return [
        render_unmapped_industry_table(frame, max_table_rows),
        render_company_review_table(max_table_rows),
        render_benefit_review_table(frame),
        render_requirement_review_table(frame),
    ]
\end{lstlisting}

\begin{itemize}
  \item \textbf{Unmapped industries}: các \texttt{job\_industry\_unmapped} chưa
    khớp taxonomy — cần cập nhật Google Sheets.
  \item \textbf{Company review}: kết quả fuzzy matching từ
    \texttt{clusters\_review.csv} (đọc từ Google Sheets hoặc file local).
  \item \textbf{Benefit review}: mẫu \texttt{BENEFIT\_SAMPLE\_SIZE = 50} benefit
    chưa map.
  \item \textbf{Requirement review}: mẫu \texttt{REQUIREMENT\_SAMPLE\_SIZE = 50}
    requirement chưa map.
\end{itemize}

\subsection{Infrastructure monitoring}
\label{subsec:infra-monitoring}

Stack Prometheus + Grafana giám sát Airflow và Supabase, cấu hình trong
file \texttt{docker-compose.yaml}:

\begin{lstlisting}[style=yamlstyle, caption={Monitoring stack — 4 services}]
services:
  caddy:        # Reverse proxy + basic_auth + TLS
    image: caddy:alpine
    ports: ["80:80", "443:443"]
  prometheus:   # Scrape Airflow StatsD + Supabase Metrics API
    image: prom/prometheus:v3.12.0
  grafana:      # Dashboards from JSON files (provisioned)
    image: grafana/grafana:11.6
  dashboards:   # nginx serving business/reports/*.html
    image: nginx:alpine
\end{lstlisting}

\subsubsection{Prometheus scrape targets}

File \texttt{prometheus.yml}
scrape 2 nguồn: Airflow statsd-exporter (metrics DAG/task) và Supabase Metrics
API (Postgres, CPU, RAM, disk):

\begin{lstlisting}[style=yamlstyle, caption={Prometheus — scrape Airflow + Supabase}]
scrape_configs:
  - job_name: airflow
    static_configs:
      - targets: ["${AIRFLOW_STATSD_EXPORTER_TARGET}"]

  - job_name: supabase
    scrape_interval: 60s
    metrics_path: /customer/v1/privileged/metrics
    scheme: https
    basic_auth:
      username: service_role
      password: ${SUPABASE_METRICS_SECRET_KEY}
    static_configs:
      - targets: ["${SUPABASE_PROJECT_REF}.supabase.co:443"]
\end{lstlisting}

\subsection{Reverse proxy (Caddy)}
\label{subsec:caddy}

File \texttt{Caddyfile} cung cấp single-login
entry point: Caddy owning auth, backend services private trên Docker network.

\begin{lstlisting}[style=bashstyle, caption={Caddyfile — reverse proxy + basic\_auth}]
{$MONITORING_DOMAIN:localhost} {
    basic_auth {
        {$MONITORING_AUTH_USER} {$MONITORING_AUTH_PASSWORD_HASH}
    }

    redir / /business/ 308
    redir /grafana /grafana/ 308

    handle_path /business/* {
        reverse_proxy dashboards:80
    }

    handle /grafana/* {
        reverse_proxy grafana:3000
    }
}
\end{lstlisting}

Caddy tự động cấp TLS (Let's Encrypt) khi \texttt{MONITORING\_DOMAIN} là domain
thật. Password hash sinh bằng \texttt{caddy hash-password}.

\subsection{Lưu ý quan trọng}
\label{subsec:gotchas}

\begin{longtable}{p{5cm} p{9cm}}
\toprule
\textbf{Vấn đề} & \textbf{Giải thích} \\
\midrule
\endhead
\texttt{DASHBOARD\_MOUNT} scoped & DAG mount \texttt{reports/} subdir vào container lakehouse-crawler, HTML output persist ra host sau khi container auto-remove. \\
Chart Altair $\to$ JSON & Chart serialize thành JSON nhúng trong \texttt{<div>} + \texttt{<script>}, không cần server-side render. \\
Grafana anonymous viewer & \texttt{GF\_AUTH\_ANONYMOUS\_ENABLE=true}, auth do Caddy xử lý ở proxy layer. \\
Supabase Metrics API & Cần \texttt{SUPABASE\_PROJECT\_REF} + \texttt{SUPABASE\_METRICS\_SECRET\_KEY} trong \texttt{.env}; scrape mỗi 60s. \\
StatsD $\to$ Prometheus & Airflow emit StatsD $\to$ statsd-exporter (Module~4) $\to$ Prometheus scrape. Cần network thông giữa 2 docker-compose stack. \\
Company review từ Google Sheets & \texttt{silver\_dashboard.py} đọc \texttt{clusters\_review.csv} từ Google Sheets trước, fallback file local. \\
\bottomrule
\end{longtable}

\subsection{Lệnh chạy}
\label{subsec:commands}

\begin{lstlisting}[style=bashstyle, caption={Khởi động monitoring stack}]
cd src/monitoring_layer
docker compose up -d

# Access via Caddy (single login)
# http://localhost/business/   -> Bronze + Silver dashboards
# http://localhost/grafana/    -> Grafana (Airflow + Supabase)
\end{lstlisting}

\begin{lstlisting}[style=bashstyle, caption={Chạy business dashboard thủ công}]
# Bronze dashboard
python -m src.monitoring_layer.business.bronze_dashboard

# Silver dashboard with date range
python -m src.monitoring_layer.business.silver_dashboard \
  --from_date 2026-06-01 --to_date 2026-06-15

# Output: src/monitoring_layer/business/reports/*.html
\end{lstlisting}
