Hạ tầng Orchestrator dùng chung

docker-compose Airflow

apache/airflow:2.10.3, LocalExecutor, postgres:16 metadata, statsd-exporter sidecar, mount /var/run/docker.sock.

COMMON_DOCKER_KWARGS

image=lakehouse-crawler:latest, auto_remove="success", shm_size=2GB, docker_url=unix:///var/run/docker.sock.

COMMON_MOUNTS

bind temp_data (append JSONL), .env (read-only), business/reports (EFS) — container auto-remove vẫn giữ output.

SITE_CONFIGS

map site → crawl_module / validate_module / bronze_source / silver_module / supabase_site; 3 site topcv, vietnamworks, itviec.

Orchestrator thuần

Airflow không import module crawler/storage; mọi business logic chạy trong container lakehouse-crawler qua DockerOperator — tách biệt rõ ràng điều phối và thực thi.

DockerOperator + docker.sock

Airflow mount socket Docker của host để spawn sibling container cùng image pipeline, bind code + env + temp — chạy đúng phiên bản code trong repo.

Per-site vs all-site

crawl / silver / validate tách theo site (cách ly lỗi một trang không ảnh hưởng trang khác); supabase / gold / dashboard gộp tất cả site trong một DAG.

TriggerDagRunOperator

crawl DAG wait_for_completion=False trigger validate_bronze_{site} — tách rời chuỗi, crawl không phải chờ validate xong mới kết thúc.

DAG params overridable

keyword, max_pages, from_date, to_date là DAG params, override qua UI "Trigger DAG w/ config" JSON mà không cần sửa code.

Schedule + retry

crawl 3h (staggered 0/15/30), silver 8h, supabase/gold 6h, dashboard midnight; catchup=False, retries=1, retry_delay=5m — không backfill, tự thử lại một lần.
